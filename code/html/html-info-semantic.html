Two structures of HTML

When coding with HTML, we use two primary structures: non-semantic and semantic. These structures differ in how they convey information to the browser.

Non-Semantic HTML

Non-semantic elements, like <div> and <span>, do not provide information about the content they contain. They are primarily used to group content together without adding meaning.

<html>
...
<div> ... <div>
<div> ... <div>
<div> ... <div>
...
</html>

Semantic HTML

Semantic elements clearly describe their meaning in a way that both the browser and developers can understand. These elements, such as <header>, <article>, <footer>, and <nav>, convey the purpose of the content contained within them.

<html>
...
<header>
<nav> ... </nav>
</header>
<article>
<figure> ... </figure>
</article>
<footer> ... </footer>
...
</html>

Advantages of Using Semantic HTML

Accessibility

  • Semantic HTML enhances accessibility by making webpages more navigable for users with disabilities. Screen readers and other assistive technologies can better interpret and present the content.
  • It also improves accessibility on mobile devices, ensuring a more consistent and understandable user experience.
  • Introduction to Web Accessibility
  • MDN Web Docs on Accessibility

Search Engine Optimization ❨SEO❩

  • Semantic elements help search engines understand the structure and content of a webpage, which can improve search engine rankings.
  • This increases the visibility and visitor count of a website, as search engines can better identify and prioritize important content.
  • Google SEO Starter Guide
  • MDN Web Docs on SEO

Readability and Maintainability

  • Semantic HTML makes the source code more readable and easier to maintain for developers. It provides a clear structure and meaning, simplifying the development process.
  • Just as clearly marked aisles in a store help shoppers find items quickly, semantic HTML helps developers navigate and understand the code more efficiently.
  • MDN Web Docs on HTML5 Semantic Elements

Combining Structures

While it is possible to mix semantic and non-semantic elements, using semantic HTML exclusively is recommended due to its numerous benefits. Semantic elements improve code clarity, accessibility, and SEO, making them superior to non-semantic elements.

In summary, opting for semantic HTML whenever possible enhances both the user experience and the development process, resulting in more accessible, search-engine-friendly, and maintainable web pages.
