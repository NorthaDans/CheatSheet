Formatting tags

Text formatting tags are used to emphasize, style, and differentiate content in various ways.
These tags play a crucial role in enhancing the readability and visual appeal of web pages.

Name Tag Explanation
Strong <strong> Renders text with a strong emphasis, typically displayed as bold. This tag is used to indicate that the text has strong importance.
Bold <b> Renders text in bold without implying any extra importance. It's often used for stylistic purposes rather than semantic emphasis.
Emphasis <em> Renders text with an italic emphasis, usually indicating stress or importance in the context of the surrounding text.
Italic <i> Renders text in italics, often used for styling text, such as book titles, technical terms, or phrases in a different language.
Big <big> Increases the size of the text, making it more prominent and noticeable compared to the surrounding content.
Small <small> Decreases the size of the text, making it less prominent and often used for disclaimers, fine print, or side comments.
Sup <sup> Renders text as superscript, positioning it slightly above the normal text line. This is often used for footnotes, mathematical exponents, and other notations.
Sub <sub> Renders text as subscript, positioning it slightly below the normal text line. Commonly used in chemical formulas and mathematical expressions.
Ins <ins> Represents inserted text, which is usually underlined to show that it has been added to the document. This tag is useful for tracking changes.
Deleted <del> Represents deleted text, often displayed with a strikethrough to show that it has been removed from the document. This is also useful for tracking changes.
Mark <Mark> Highlights text, marking it with a background color to indicate its importance or relevance. This is particularly useful for drawing attention to specific parts of the text.
Abbreviation <abbr> Represents an abbreviation or acronym, providing the full expansion of the term when hovered over in some browsers.
Cite <cite> Represents a citation or reference to a source.
This tag is often used for titles of books, articles, or other works.
Code <code> Displays a fragment of computer code, usually in a monospace font. This tag is often used for programming code or commands.
Sample output <samp> Represents sample output from a computer program. This tag is useful for displaying text that shows the result of code execution.
Define <dfn> Represents the defining instance of a term. When the term is first introduced and defined in a document, this tag is used.
Keyboard input <kbd> Represents user input, typically from a keyboard. This tag is useful for displaying keypresses or text that the user should type.
Predefined <pre> Preserves whitespace and displays text in a fixed-width font. This tag is often used for preformatted text, such as code blocks or ASCII art.
Inline quotation <q> Represents a short inline quotation. Browsers typically render the content inside quotation marks.
Underline <u> Underlines text, often used for stylistic purposes rather than conveying emphasis or importance.
Variable <var> Represents a variable in a mathematical expression or programming context. This tag is typically rendered in italic.

Table tags

Tables in HTML are used to organize and display data in a structured format, making it easier to read and understand.

Name Tag Explanation
Table <table> The container element for all tabular data. It defines the structure and layout of the table.
Table head <thead> Defines the header section of the table, typically containing column headings. It is a child of the <table> element and often appears before <tbody>.
Table body <tbody> Groups the body content in a table. This tag helps in organizing and managing long tables by sectioning off the main content area.
Table footer <tfoot> Groups the footer content in a table, usually containing summary information like totals or averages. It is a child of the <table> element and can be placed before or after <tbody>.
Table row <tr> Represents a row in a table. It is a child of <table>, <thead>, <tbody>, or <tfoot>, and contains the table cells.
Table heading <th> Defines a header cell in a table. It is a child of a <tr> element and is used to provide titles for the columns or rows.
Table data <td> Defines a standard data cell in a table. It is a child of a <tr> element and contains the actual data for the table.

Form

The <form> attribute is used to connect and send information. The tag needs to be supplied with an instruction on where to send data and which http requests to make.

<form
action="/example.html"
method="POST">
</form>

In the above example the data will be sent to example.html via a POST method. The action thus informs where the request is due to be sent and method includes a http verb that is included in the http request which instructs which http request to make to deliver the data.

It would be helpful to provide a header so that users know what this <form> is about. We could also add a paragraph to provide even more detail.

Let's see an example of this in code:

<form action="/example.html" method="POST">
<h1> Creating a form </h1>

Input

The <input> tag creates an input field that users can type data into. It also has a type attribute. This decides how it renders on the screen and what kind of data it can accept. The type attribute is set by default to text which renders a text box that users type into.

The tag strictly requires that it contains a name attribute. If it is missing, the data will not be sent once submitted.
The value of the value attribute and of the name are both sent in text form upon submission.

<form
action ="/example.html"
method ="POST" >
<input
type ="text"
name ="first-text-field" >
</form>
When you type into the text box, the entered text will be assigned to the `value` attribute of the input field.
Upon submission, the form will send a POST request to the specified URL. The data sent with this request will include the name-value pair defined by the input field's `name` attribute `first-text-field` and the user's input as its value.
This process allows the server to receive and process the user's input.

We could also assign a default value for the value attribute so that users have a pre-filled text field when they first see the rendered form like so:

<form
action ="/example.html"
method ="POST" >
<input
type ="text"
name ="first-text-field"
value ="already pre-filled" >
</form>

The data sent with this request will include the name-value pair defined by the input field's name attribute `first-text-field` and the user's input as its value.
This process allows the server to receive and process the user's input.

Introduction to HTML Form Validation

Validation is the process of checking user-provided data against the required criteria. It ensures that the data entered is correct and secure before granting access or processing further.

Types of Validation

Server-Side Validation:
  • This occurs when data is sent to a server for validation.
  • Example: When you enter your username and password on a login page, the form data is sent to a server. The server checks if the provided credentials match the stored data.
  • MDN Web Docs - Server-side validation
Client-Side Validation:
  • This happens in the user's browser before the data is sent to the server.
  • Different browsers implement client-side validation in various ways, but the goal is the same: to validate data before it reaches the server.
  • MDN Web Docs - Client-side Validation

Benefits of Client-Side Validation

  • Efficiency: HTML5's built-in client-side validation can save time by reducing the need for server communication. Immediate feedback is provided, so users can correct errors without waiting for a server response.
  • Security: By validating data on the client side, we can filter out obviously incorrect or malicious input before it reaches the server, adding an extra layer of security.
  • User Experience: Users receive instant feedback on specific fields, improving their experience by allowing them to correct errors in real time rather than resubmitting the entire form.
  • MDN Web Docs - HTML5 Constraint Validation

By understanding and implementing these types of validation, you can enhance both the security and usability of your web applications.

Matchig to a pattern

In addition to checking the length of a text, we could also add a validation to check how the text was provided. For cases when we want user input to follow specific guidelines, we use the pattern attribute and assign it a regular expression, or regex. Regular expressions are a sequence of characters that make up a search pattern.
If the input matches the regex, the form can be submitted.

Let's say we wanted to check for a valid credit card number (a 14 to 16 digit number). We could use the regex: [0-9]{14,16} which checks that the user provided only numbers and that they entered at least 14 digits and at most 16 digits.

<form
action ="/example.html"
method ="POST"
<label for ="payment" >
Credit Card Number ❨no spaces❩:
</label>
<br>
<input
id ="payment"
name ="payment"
type ="text"
required
pattern ="[0-9]{14,16}" >
<input
type ="submit"
value ="Submit" >
</form>

With the pattern in place, users can't submit the <form> with a number that doesn't follow the regex.
